# 贡献指南

欢迎 Issue、文档、测试和代码贡献。

1. 先搜索现有 Issue，较大改动请先讨论边界。
2. 从 `main` 创建短期分支，保持改动聚焦。
3. 不得提交密钥、生产地址、用户数据、数据库、日志或来源不明素材。
4. Go 改动运行受影响包测试；Web 改动运行 `npm test` 与 `npm run build`；小程序改动运行仓库规定的 clean build 序列。
5. 提交必须签署 DCO：`git commit -s`。
6. PR 描述应包含动机、改动、验证和安全/兼容性影响。

提交即表示你有权按 AGPL-3.0 提供该贡献，并同意 [DCO](DCO)。
