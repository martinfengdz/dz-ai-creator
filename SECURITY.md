# 安全政策

## 支持范围

安全修复面向最新发布版本和 `main`。旧版本可能要求先升级后验证。

## 报告漏洞

请优先使用 GitHub Private Vulnerability Reporting。若仓库尚未启用该功能，请通过仓库所有者在 GitHub 个人资料中公布的安全联系方式私下报告。

报告请包含受影响版本、复现步骤、影响、最小化证据和建议缓解措施。不要提交真实用户数据、访问令牌、私钥或可直接利用的公开 Issue。

我们目标在 3 个工作日内确认收到报告，在 7 个工作日内给出初步分级与处置计划。修复公开前请给维护者合理的协调时间。

## 凭据事件

凭据一旦进入提交、日志、截图或共享文件，应立即在供应商侧作废并重新签发；仅从 Git 中删除并不能恢复安全性。历史泄露凭据不得加入 Gitleaks baseline 或白名单。
